# The Original Word hub is a morpheme-grained, externally-keyed bridge with a separable lexicon

Phase 3 adds the fine, word-grain bridge between translations — the **Original Word hub** + **interlinear alignment** — and the Strong's/lemma **lexicon**. [ADR-0001](0001-three-layer-anchor-model.md) locked that cross-translation relations route through this hub by an optional Original-Word id (`ow_id`), and that id already sits on four tables (`token.ow_id`, `mark.target_ow_id`, `note.pin_ow_id`, `binding.anchor_ow_id`), null since Phase 1. This ADR fixes the *storage shape* the hub + lexicon land in — what fills that seam, and how the corpus, hub, and lexicon reference each other. Resolved across a focused grill (Q1–Q9) and recorded in [`schema.dbml`](../../schema.dbml).

The decisions:

1. **`ow_id` is an opaque external per-occurrence word-id** (MACULA / STEPBible / OSHB class), not a self-minted coordinate (Q1). Stability is outsourced to a maintained corpus; alignment import becomes a straight join instead of a fuzzy match; licensing is redistributable (the NASB lesson, applied to reference data). It is **never parsed** → remappable via a translation table if we ever swap source corpora.
2. **The hub is morpheme-grained.** An `original_word` is a lexical *segment*-occurrence — a whole word in Greek, a **morpheme** in Hebrew — because the external corpora mint ids at morpheme grain and alignment targets morphemes ("in" → the Hebrew preposition). A **written word** is regrouped via `segment_of` + `segment_index`; Greek words are trivially single-segment (Q7). One uniform entity means alignment, gloss, mark, and morphology all target an `ow_id` with no morpheme-special path.
3. **`token.ow_id` is the denormalized head; `interlinear_alignment` is the M:N truth** (Q2, Q9). The single column answers "the one Original Word this Token primarily maps to" (portability + fast path); the junction carries the full many-to-many for interlinear rendering. The junction is **bare** — `(token_id, original_word_id)` — with relation-type / confidence / primary-flag left additive (no source for them yet).
4. **Three tiers join by stable string keys; FK only within a tier** (Q3). `corpus → hub → lexicon`: anything crossing a boundary (`token.ow_id`, `alignment.original_word_id`, `original_word.lemma_id`/`strong_id`/`parse_code`/`segment_of`) is a string key with **no FK**. The only real FK is corpus-internal (`alignment.token_id → token.id`, regenerated on ingest like `block_id`). Cross-tier integrity is an **ingest-time validation pass**, not a runtime FK. So physical packaging (one bundled `.sqlite` vs separate attachable lexicon/hub files) stays reversible — the shapes don't change.
5. **The hub denormalizes both addresses inline** (Q4). Each `original_word` carries the **canonical** `(book, chapter, verse)` (the bridge to the reading corpus) *and* the **native** Hebrew/Greek `(chapter, verse)` (fidelity), plus `position`. Read-only reference data, so the hot interlinear/bridge path never routes through a split/merge-capable versification map.
6. **`lemma` (spine) + `strongs` (launch dictionary) now; the multi-lexicon layer deferred** (Q5). `lemma` and Strong's are separate identifier spaces, both referenced from the hub. `lexicon_entry` (BDB/Thayer), `root` (Hebrew), and `semantic_domain` (Louw–Nida) are purely additive later tables that hang off `lemma` — zero migration.
7. **Morphology is a raw code on the word + a code-keyed decode table** (Q6). `original_word.parse_code` is the source of truth; `morphology(code → features)` is the small queryable decoded view — not per-word flat columns.
8. **Gloss is inline on the hub; per-translation gloss is rejected** (Q8). One editorial interlinear gloss (`gloss` + `gloss_prefix`/`gloss_suffix`) per Original Word. Reverse-interlinear (a reading translation's words under the original) is *derived* by walking alignment, never stored.

## Considered options

- **Self-minted original-text coordinate as `ow_id`** — rejected: re-segmenting Hebrew renumbers positions, and `ow_id` is migration-fatal (every word-grain Mark/Note created since Phase 2 would orphan). An external maintained id won't renumber under us.
- **Whole-word `original_word` + child `morpheme` table** — rejected: fights the source id grain (ids are minted per morpheme) and forces morpheme-aware code paths for alignment/gloss/mark.
- **FK everything into one file** — rejected: welds the lexicon to the corpus, so re-sourcing Strong's or shipping a translation pack independently becomes a coordinated migration.
- **1:1 Token↔Original-Word, no alignment table** — rejected: loses compounds (`G1 G2` on one token) and phrase glosses; real alignment is many-to-many.
- **Per-reading-translation gloss rows** — rejected: derivable from alignment + `token.text`; storing them duplicates data.
- **Generic multi-lexicon `lexicon_entry` at launch** — rejected: Strong's is keyed by its own number, not by lemma, so forcing it through a lemma-keyed table is awkward and risks mis-keying with no second source yet to justify it.
- **Pre-decoded morphology columns per word** — rejected: a wide sparse two-language table (Greek case vs Hebrew stem), and brittle (the old `Pos` enum was a 3-stem/1-person stub).

## Consequences

- The hub + lexicon version **independently** of the corpus and of the annotation layer. A corpus re-ingest leaves the only cross-tier references (string keys) intact; the one real FK (`alignment → token`) regenerates with the corpus, exactly like `block_id`.
- The Phase-1/2 `ow_id` seam is now **wired with zero migration** — word-grain Markup gains true cross-translation word portability the moment the hub populates, exactly the "verse-grain fallback now, word-grain later" the prototype validated ([ADR-0005](0005-four-product-phases-design-all-build-in-order.md)).
- **No DB-enforced integrity across tiers** — an ingest-time validation pass must assert every `ow_id` / `strong_id` / `lemma_id` / `parse_code` resolves. This is the price of separability. **It must also assert `token.ow_id ∈ interlinear_alignment(token)` (#10):** the "head" is not derivable from the bare junction (no `is_primary` flag yet), so the ingester sets it out-of-band from the source's primary mapping, and only the validation pass catches a head that drifts from the alignment. Add `is_primary` to the junction additively if a source ever supplies it.
- **"Lemma is the spine" depends on sourcing (#11).** `original_word.lemma_id` assumes the chosen corpus supplies a lemma id per occurrence (MACULA/OSHB do — confirm). If a source ships Strong's but no lemma, `lemma_id` is null at launch and occurrence statistics hang off `strong_id` until a lemma source is added — additive, no migration.
- **"Original Word" now denotes a morpheme-occurrence** ([CONTEXT.md](../../CONTEXT.md) sharpened). Downstream UI that says "word" must group segments for Hebrew (`segment_of`).
- **Sourcing is the open risk, not the schema.** A redistributable interlinear corpus (MACULA / STEPBible / OSHB+MorphGNT) must be chosen and license-checked — the same discipline that keeps NASB out of public bundles.
- **Versification split/merge stays deferred.** The hub sidesteps it via inline dual addresses; reading translations with clean splits/merges remain a separate later item already noted in `schema.dbml`.
